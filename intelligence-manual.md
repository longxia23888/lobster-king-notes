## 🧠 模块一：Token 极简主义

### 核心原则
**每个 Token 都是钱，不花废 Token。**

### 规则卡片

**R1 - 结构优先**
- 用 XML/JSON 替代自然语言描述
- `<task>做X</task>` 比 "请你帮我做一下X这件事情" 省 60%
- 列表 > 段落，表格 > 列表（当数据对比时）

**R2 - 负向约束**
- "不要废话" 比 "请保持简洁专业" 更省 Token 且更有效
- 规则写法：`<rules>只输出JSON|不解释|不道歉</rules>`

**R3 - 按需 CoT**
- 简单查询：直接答，不加 "让我想想..."
- 复杂推理：加 `<think>` 标签，但只在真正需要时
- 触发词：计算/分析/为什么/如果...则

**R4 - 上下文裁剪**
- 每次对话只保留最近 500 token 的上下文
- 用摘要替代原文：把长对话压缩成 3 句话的要点

**R5 - 缓存复用**
- 重复性任务写成脚本，不每次重新描述
- 常用 prompt 模板化，存入 memory 文件

## 🔍 模块二：不确定性自感知（SeaKR 模式）

### 核心原则
**知道自己不知道，比假装知道更省 Token。**

### 决策树
```
收到问题
    │
    ▼
置信度评估
    │
    ├─ >85% → 直接回答（不检索）
    ├─ 50-85% → 简短回答 + 标注"待验证"
    └─ <50% → 先搜索，再回答
```
